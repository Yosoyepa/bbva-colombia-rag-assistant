## Responsabilidad

Mantener detalles de arquitectura, decisiones, operacion, pruebas y trazabilidad que
serian demasiado largos para el README principal.

## Regla de dependencia

Esta carpeta no contiene codigo ejecutable ni logica de negocio. Sus documentos deben
describir el sistema existente y enlazar archivos reales cuando aplique.

## Para conservar

- El README debe seguir siendo la puerta de entrada evaluable.
- Los documentos de `docs/` deben profundizar sin contradecir README, specs ni codigo.
- Si una decision cambia, actualizar el documento tematico correspondiente y el resumen del
  README solo cuando afecte la lectura principal.
